   40: =head1 DESCRIPTION
   41: 
   42: C<Params::Get> exports a single function, C<get_params>, which accepts a
   43: caller's argument list (or a reference to it) in any of the common Perl
   44: calling conventions and returns a unified hash-ref.  Library authors can
   45: write one normalisation call at the top of every public method rather than
   46: hand-rolling the same conditional chains in each one.
   47: 
   48: When combined with L<Params::Validate::Strict> and L<Return::Set> you can
   49: formally specify and enforce the input and output contracts of every method.
   50: 
   51: =head1 SYNOPSIS
   52: 
   53:     use Params::Get qw(get_params);
   54:     use Params::Validate::Strict;
   55: 
   56:     sub where_am_i {
   57:         my $params = Params::Validate::Strict::validate_strict({
   58:             args   => get_params(undef, \@_),
   59:             schema => {
   60:                 latitude  => { type => 'number', min => -90,  max =>  90 },
   61:                 longitude => { type => 'number', min => -180, max => 180 },
   62:             },
   63:         });
   64:         printf "You are at %s, %s\n",
   65:             $params->{latitude}, $params->{longitude};
   66:     }
   67: 
   68:     where_am_i(latitude => 0.3, longitude => 124);
   69:     where_am_i({ latitude => 3.14, longitude => -155 });
   70: 
   71: =head1 METHODS
   72: 
   73: =head2 get_params
   74: 
   75: Parse the argument list passed to a subroutine and return a unified hash-ref
   76: regardless of the calling convention used.  Supported conventions:
   77: 
   78: =over 4
   79: 
   80: =item * Single hash-ref: C<foo({ a =E<gt> 1 })>
   81: 
   82: =item * Named key/value pairs: C<foo(a =E<gt> 1, b =E<gt> 2)>
   83: 
   84: =item * Single scalar with a default key: C<foo('US')> given C<get_params('country', @_)>
   85: 
   86: =item * Array-ref shorthand: C<foo(\@_)> inside the callee
   87: 
   88: =item * Mandatory positional argument plus an options hash-ref:
   89: C<Obj-E<gt>new($val, { opt =E<gt> 1 })>
   90: 
   91: =item * Scalar-ref: C<foo(\'text')> -- dereferenced automatically
   92: 
   93: =item * Blessed object or CODE ref: mapped under C<$default>
   94: 
   95: =item * Array-ref of positional key names as C<$default>:
   96: C<get_params([qw(name age)], @_)>
   97: 
   98: =back
   99: 
  100: =head3 ARGUMENTS
  101: 
  102: =over 4
  103: 
  104: =item C<$default> (scalar string, arrayref of strings, or C<undef>)
  105: 
  106: Controls how a single non-hash argument is interpreted:
  107: 
  108: =over 8
  109: 
  110: =item * B<string> -- used as the key name when a lone scalar, ref, or object
  111: is received.
  112: 
  113: =item * B<arrayref of strings> -- positional key names; the I<n>th argument
  114: is mapped to the I<n>th name.  Extra arguments are silently discarded;
  115: missing arguments produce C<undef> values.
  116: 
  117: =item * B<undef> -- no default key; the caller must pass named pairs or a
  118: single hash-ref.  An empty argument list returns C<undef>.
  119: 
  120: =back
  121: 
  122: =item C<@args>
  123: 
  124: The caller's argument list, passed either as a flat list (C<@_>) or as a
  125: reference to the array (C<\@_>).  Both forms are accepted transparently.
  126: 
  127: =back
  128: 
  129: =head3 RETURNS
  130: 
  131: A hash-ref on success, or C<undef> when C<$default> is C<undef> and no
  132: arguments are provided.
  133: 
  134: =head3 SIDE EFFECTS
  135: 
  136: Croaks from the caller's frame on programming errors (wrong calling
  137: convention, non-ARRAY ref passed as C<$default>).  Confesses with a full
  138: stack trace when C<$default> is defined but zero arguments are received,
  139: because that almost always indicates a programming error.
  140: 
  141: =head3 API SPECIFICATION
  142: 
  143:     Input:
  144:         $default : Str | ArrayRef[Str] | Undef
  145:         @args    : Any*     # or a single ArrayRef containing Any*
  146: 
  147:     Output (Return::Set):
  148:         HashRef | Undef

  305: =head1 LIMITATIONS
  306: 
  307: =over 4
  308: 
  309: =item B<Single empty arrayref cannot be distinguished from C<\@_> of an empty list>
  310: 
  311: When the caller does C<foo([])> and the callee uses C<get_params('key', @_)>,
  312: the C<@_> list is C<([])> -- one element, an arrayref.  The function
  313: interprets the lone arrayref as a C<\@_> passthrough, unwraps it to an empty
  314: list, and then croaks because C<$default> is defined but there are zero
  315: arguments.  Workaround: pass the value as a named pair
  316: (C<key =E<gt> []>) or ensure the callee always uses C<\@_>.
  317: 
  318: =item B<Single hash ref always bypasses C<$default> key naming>
  319: 
  320: C<get_params('config', { a =E<gt> 1 })> returns C<{ a =E<gt> 1 }>, not
  321: C<{ config =E<gt> { a =E<gt> 1 } }>.  The fast path fires before C<$default>
  322: is inspected.  To store a hash ref under a default key, pass it as a named
  323: pair: C<get_params('config', config =E<gt> { a =E<gt> 1 })>.
  324: 
  325: =item B<No mechanism to mark a C<$default> argument as optional>
  326: 
  327: When C<$default> is a string and zero arguments are received, the function
  328: always confesses.  There is no way to express I<"accept zero args and return
  329: undef gracefully">.
  330: 
  331: =item B<Duplicate keys in a flat list silently overwrite; last value wins>
  332: 
  333: C<get_params(undef, foo =E<gt> 1, foo =E<gt> 2)> returns C<{ foo =E<gt> 2 }>
  334: with no warning.  If an attacker controls part of the argument list, a
  335: later duplicate key can silently override an earlier sanitised value.
  336: Detect and reject duplicate keys in the validation layer (e.g.
  337: L<Params::Validate::Strict>) rather than relying on C<get_params> to catch
  338: them.
  339: 
  340: =item B<Positional-names C<$default> silently discards extra arguments>
  341: 
  342: C<get_params([qw(a b)], 1, 2, 3)> returns C<{ a =E<gt> 1, b =E<gt> 2 }>
  343: and ignores C<3>.  If strict arity is required, validate the returned hash
  344: with L<Params::Validate::Strict>.
  345:

  387: =head3 get_params
  388: 
  389:     Let D = default key (Str | [Str*] | undef), A = argument tuple.
  390: 
  391:     get_params : D x A* -> HashRef | Undef
  392: 
  393:     -- Fast path (fires before D is inspected -- see LIMITATIONS)
  394:     get_params(D, h)            == h             when |A|=1, h:HashRef
  395: 
  396:     -- Positional-names default
  397:     get_params([n1..nk], v*)    == {ni -> vi}    i in 1..k, vi = undef when missing
  398: 
  399:     -- Scalar default, single arg
  400:     get_params(d, s)            == {d -> s}      d:Str, s:Scalar
  401:     get_params(d, a)            == {d -> a}      d:Str, a:ArrayRef
  402:     get_params(d, \s)           == {d -> s}      d:Str (scalarref dereferenced)
  403:     get_params(d, c)            == {d -> c}      d:Str, c:CodeRef
  404:     get_params(d, o)            == {d -> o}      d:Str, o:BlessedObject
  405: 
  406:     -- Mandatory-positional + options-hashref
  407:     get_params(d, v, {k->w..})  == {d->v, k->w..}    non-empty opts
  408:     get_params(d, d, {k->w..})  == {d -> {k->w..}}   first arg IS the key name
  409: 
  410:     -- Named pairs
  411:     get_params(undef, k1,v1..)  == {ki -> vi}    when |A| is even
  412: 
  413:     -- Empty / error
  414:     get_params(undef)           == undef
  415:     get_params(d)               => confess       d:Str (missing required arg)
  416:     get_params(D, odd-list)     => croak
