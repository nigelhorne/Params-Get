   42: =head1 DESCRIPTION
   43: 
   44: C<Params::Get> exports a single function, C<get_params>, which accepts a
   45: caller's argument list (or a reference to it) in any of the common Perl
   46: calling conventions and returns a unified hash-ref.  Library authors can
   47: write one normalisation call at the top of every public method rather than
   48: hand-rolling the same conditional chains in each one.
   49: 
   50: When combined with L<Params::Validate::Strict> and L<Return::Set> you can
   51: formally specify and enforce the input and output contracts of every method.
   52: 
   53: =head1 SYNOPSIS
   54: 
   55:     use Params::Get qw(get_params);
   56:     use Params::Validate::Strict;
   57: 
   58:     sub where_am_i {
   59:         my $params = Params::Validate::Strict::validate_strict({
   60:             args   => get_params(undef, \@_),
   61:             schema => {
   62:                 latitude  => { type => 'number', min => -90,  max =>  90 },
   63:                 longitude => { type => 'number', min => -180, max => 180 },
   64:             },
   65:         });
   66:         printf "You are at %s, %s\n",
   67:             $params->{latitude}, $params->{longitude};
   68:     }
   69: 
   70:     where_am_i(latitude => 0.3, longitude => 124);
   71:     where_am_i({ latitude => 3.14, longitude => -155 });
   72: 
   73: =head1 METHODS
   74: 
   75: =head2 get_params
   76: 
   77: Parse the argument list passed to a subroutine and return a unified hash-ref
   78: regardless of the calling convention used.  Supported conventions:
   79: 
   80: =over 4
   81: 
   82: =item * Single hash-ref: C<foo({ a =E<gt> 1 })>
   83: 
   84: =item * Named key/value pairs: C<foo(a =E<gt> 1, b =E<gt> 2)>
   85: 
   86: =item * Single scalar with a default key: C<foo('US')> given C<get_params('country', @_)>
   87: 
   88: =item * Array-ref shorthand: C<foo(\@_)> inside the callee
   89: 
   90: =item * Mandatory positional argument plus an options hash-ref:
   91: C<Obj-E<gt>new($val, { opt =E<gt> 1 })>
   92: 
   93: =item * Scalar-ref: C<foo(\'text')> -- dereferenced automatically
   94: 
   95: =item * Blessed object or CODE ref: mapped under C<$default>
   96: 
   97: =item * Array-ref of positional key names as C<$default>:
   98: C<get_params([qw(name age)], @_)>
   99: 
  100: =back
  101: 
  102: =head3 ARGUMENTS
  103: 
  104: =over 4
  105: 
  106: =item C<$default> (scalar string, arrayref of strings, or C<undef>)
  107: 
  108: Controls how a single non-hash argument is interpreted:
  109: 
  110: =over 8
  111: 
  112: =item * B<string> -- used as the key name when a lone scalar, ref, or object
  113: is received.
  114: 
  115: =item * B<arrayref of strings> -- positional key names; the I<n>th argument
  116: is mapped to the I<n>th name.  Extra arguments are silently discarded;
  117: missing arguments produce C<undef> values.
  118: 
  119: =item * B<undef> -- no default key; the caller must pass named pairs or a
  120: single hash-ref.  An empty argument list returns C<undef>.
  121: 
  122: =back
  123: 
  124: =item C<@args>
  125: 
  126: The caller's argument list, passed either as a flat list (C<@_>) or as a
  127: reference to the array (C<\@_>).  Both forms are accepted transparently.
  128: 
  129: =back
  130: 
  131: =head3 RETURNS
  132: 
  133: A hash-ref on success, or C<undef> when C<$default> is C<undef> and no
  134: arguments are provided.
  135: 
  136: =head3 SIDE EFFECTS
  137: 
  138: Croaks from the caller's frame on programming errors (wrong calling
  139: convention, non-ARRAY ref passed as C<$default>).  Confesses with a full
  140: stack trace when C<$default> is defined but zero arguments are received,
  141: because that almost always indicates a programming error.
  142:

  175: =head3 PSEUDOCODE
  176: 
  177:     1.  Fast-path: if the sole argument is a plain HASH ref, return it
  178:         immediately (fires before $default is inspected -- see LIMITATIONS).
  179: 
  180:     2.  Shift $default.  Validate: must be undef, a plain scalar, or an
  181:         ARRAY ref.  Any other ref type croaks immediately.
  182: 
  183:     3.  If $default is an ARRAY ref, map remaining @_ positionally to those
  184:         key names and return.  A single plain HASH ref is still passed
  185:         through unchanged.
  186: 
  187:     4.  Detect the \@_ calling convention: if exactly one ARRAY ref argument
  188:         remains, check the two-element (key => scalar-val) shorthand and
  189:         return immediately when it matches.  Otherwise unwrap and use the
  190:         array contents as the effective @args.
  191: 
  192:     5.  Dispatch on argument count:
  193:         0 -- confess (with stack trace) if $default is defined;
  194:              return undef otherwise.
  195:         1 -- if $default is defined, wrap the single arg under $default
  196:              (scalar, arrayref, scalarref->deref, coderef, blessed object).
  197:              Without $default: unwrap REF-of-REF, pass HASH ref through,
  198:              return empty ARRAY ref as-is.  Anything else: croak.
  199:         2 with HASH ref as arg[1]
  200:           -- Mandatory-positional + options-hashref pattern.
  201:         even N -- treat as flat key/value pairs.
  202:         odd N  -- croak.

  317: =head1 LIMITATIONS
  318: 
  319: =over 4
  320: 
  321: =item B<Single empty arrayref cannot be distinguished from C<\@_> of an empty list>
  322: 
  323: When the caller does C<foo([])> and the callee uses C<get_params('key', @_)>,
  324: the C<@_> list is C<([])> -- one element, an arrayref.  The function
  325: interprets the lone arrayref as a C<\@_> passthrough, unwraps it to an empty
  326: list, and then croaks because C<$default> is defined but there are zero
  327: arguments.  Workaround: pass the value as a named pair
  328: (C<key =E<gt> []>) or ensure the callee always uses C<\@_>.
  329: 
  330: =item B<Single hash ref always bypasses C<$default> key naming>
  331: 
  332: C<get_params('config', { a =E<gt> 1 })> returns C<{ a =E<gt> 1 }>, not
  333: C<{ config =E<gt> { a =E<gt> 1 } }>.  The fast path fires before C<$default>
  334: is inspected.  To store a hash ref under a default key, pass it as a named
  335: pair: C<get_params('config', config =E<gt> { a =E<gt> 1 })>.
  336: 
  337: =item B<No mechanism to mark a C<$default> argument as optional>
  338: 
  339: When C<$default> is a string and zero arguments are received, the function
  340: always confesses.  There is no way to express I<"accept zero args and return
  341: undef gracefully">.
  342: 
  343: =item B<Duplicate keys in a flat list silently overwrite; last value wins>
  344: 
  345: C<get_params(undef, foo =E<gt> 1, foo =E<gt> 2)> returns C<{ foo =E<gt> 2 }>
  346: with no warning.  If an attacker controls part of the argument list, a
  347: later duplicate key can silently override an earlier sanitised value.
  348: Detect and reject duplicate keys in the validation layer (e.g.
  349: L<Params::Validate::Strict>) rather than relying on C<get_params> to catch
  350: them.
  351: 
  352: =item B<Positional-names C<$default> silently discards extra arguments>
  353: 
  354: C<get_params([qw(a b)], 1, 2, 3)> returns C<{ a =E<gt> 1, b =E<gt> 2 }>
  355: and ignores C<3>.  If strict arity is required, validate the returned hash
  356: with L<Params::Validate::Strict>.
  357:

  401: =head3 get_params
  402: 
  403:     Let D = default key (Str | [Str*] | undef), A = argument tuple.
  404: 
  405:     get_params : D x A* -> HashRef | Undef
  406: 
  407:     -- Fast path (fires before D is inspected -- see LIMITATIONS)
  408:     get_params(D, h)            == h             when |A|=1, h:HashRef
  409: 
  410:     -- Positional-names default
  411:     get_params([n1..nk], v*)    == {ni -> vi}    i in 1..k, vi = undef when missing
  412: 
  413:     -- Scalar default, single arg
  414:     get_params(d, s)            == {d -> s}      d:Str, s:Scalar
  415:     get_params(d, a)            == {d -> a}      d:Str, a:ArrayRef
  416:     get_params(d, \s)           == {d -> s}      d:Str (scalarref dereferenced)
  417:     get_params(d, c)            == {d -> c}      d:Str, c:CodeRef
  418:     get_params(d, o)            == {d -> o}      d:Str, o:BlessedObject
  419: 
  420:     -- Mandatory-positional + options-hashref
  421:     get_params(d, v, {k->w..})  == {d->v, k->w..}    non-empty opts
  422:     get_params(d, d, {k->w..})  == {d -> {k->w..}}   first arg IS the key name
  423: 
  424:     -- Named pairs
  425:     get_params(undef, k1,v1..)  == {ki -> vi}    when |A| is even
  426: 
  427:     -- Empty / error
  428:     get_params(undef)           == undef
  429:     get_params(d)               => confess       d:Str (missing required arg)
  430:     get_params(D, odd-list)     => croak
