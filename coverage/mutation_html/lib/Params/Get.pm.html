   41: =head1 DESCRIPTION
   42: 
   43: C<Params::Get> exports a single function, C<get_params>, which accepts a
   44: caller's argument list (or a reference to it) in any of the common Perl
   45: calling conventions and returns a unified hash-ref.  Library authors can
   46: write one normalisation call at the top of every public method rather than
   47: hand-rolling the same conditional chains in each one.
   48: 
   49: When combined with L<Params::Validate::Strict> and L<Return::Set> you can
   50: formally specify and enforce the input and output contracts of every method.
   51: 
   52: =head1 SYNOPSIS
   53: 
   54:     use Params::Get qw(get_params);
   55:     use Params::Validate::Strict;
   56: 
   57:     sub where_am_i {
   58:         my $params = Params::Validate::Strict::validate_strict({
   59:             args   => get_params(undef, \@_),
   60:             schema => {
   61:                 latitude  => { type => 'number', min => -90,  max =>  90 },
   62:                 longitude => { type => 'number', min => -180, max => 180 },
   63:             },
   64:         });
   65:         printf "You are at %s, %s\n",
   66:             $params->{latitude}, $params->{longitude};
   67:     }
   68: 
   69:     where_am_i(latitude => 0.3, longitude => 124);
   70:     where_am_i({ latitude => 3.14, longitude => -155 });

   72: =head1 FUNCTIONS
   73: 
   74: =head2 get_params
   75: 
   76: =head3 PURPOSE
   77: 
   78: Parse the argument list passed to a subroutine and return a unified hash-ref
   79: regardless of the calling convention used.  Supported conventions:
   80: 
   81: =over 4
   82: 
   83: =item * Single hash-ref: C<foo({ a =E<gt> 1 })>
   84: 
   85: =item * Named key/value pairs: C<foo(a =E<gt> 1, b =E<gt> 2)>
   86: 
   87: =item * Single scalar with a default key: C<foo('US')> given C<get_params('country', @_)>
   88: 
   89: =item * Array-ref shorthand: C<foo(\@_)> inside the callee
   90: 
   91: =item * Mandatory positional argument plus an options hash-ref:
   92: C<Obj-E<gt>new($val, { opt =E<gt> 1 })>
   93: 
   94: =item * Scalar-ref: C<foo(\'text')> -- dereferenced automatically
   95: 
   96: =item * Blessed object or CODE ref: mapped under C<$default>
   97: 
   98: =item * Array-ref of positional key names as C<$default>:
   99: C<get_params([qw(name age)], @_)>
  100: 
  101: =back
  102: 
  103: =head3 ARGUMENTS
  104: 
  105: =over 4
  106: 
  107: =item C<$default> (scalar string, arrayref of strings, or C<undef>)
  108: 
  109: Controls how a single non-hash argument is interpreted:
  110: 
  111: =over 8
  112: 
  113: =item * B<string> -- used as the key name when a lone scalar, ref, or object
  114: is received.
  115: 
  116: =item * B<arrayref of strings> -- positional key names; the I<n>th argument
  117: is mapped to the I<n>th name.  Extra arguments are silently discarded;
  118: missing arguments produce C<undef> values.
  119: 
  120: =item * B<undef> -- no default key; the caller must pass named pairs or a
  121: single hash-ref.  An empty argument list returns C<undef>.
  122: 
  123: =back
  124: 
  125: =item C<@args>
  126: 
  127: The caller's argument list, passed either as a flat list (C<@_>) or as a
  128: reference to the array (C<\@_>).  Both forms are accepted transparently.
  129: 
  130: =back
  131: 
  132: =head3 RETURNS
  133: 
  134: A hash-ref on success, or C<undef> when C<$default> is C<undef> and no
  135: arguments are provided.
  136: 
  137: =head3 SIDE EFFECTS
  138: 
  139: Croaks from the caller's frame on programming errors (wrong calling
  140: convention, non-ARRAY ref passed as C<$default>).  Confesses with a full
  141: stack trace when C<$default> is defined but zero arguments are received,
  142: because that almost always indicates a programming error.
  143:

  148:         @args    : Any*     # or a single ArrayRef containing Any*

  150:     Output (Return::Set):

  338: =head1 LIMITATIONS
  339: 
  340: =over 4
  341: 
  342: =item B<Single empty arrayref cannot be distinguished from C<\@_> of an empty list>
  343: 
  344: When the caller does C<foo([])> and the callee uses C<get_params('key', @_)>,
  345: the C<@_> list is C<([])> -- one element, an arrayref.  The function
  346: interprets the lone arrayref as a C<\@_> passthrough, unwraps it to an empty
  347: list, and then croaks because C<$default> is defined but there are zero
  348: arguments.  Workaround: pass the value as a named pair
  349: (C<key =E<gt> []>) or ensure the callee always uses C<\@_>.
  350: 
  351: =item B<Single hash ref always bypasses C<$default> key naming>
  352: 
  353: C<get_params('config', { a =E<gt> 1 })> returns C<{ a =E<gt> 1 }>, not
  354: C<{ config =E<gt> { a =E<gt> 1 } }>.  The fast path fires before C<$default>
  355: is inspected.  To store a hash ref under a default key, pass it as a named
  356: pair: C<get_params('config', config =E<gt> { a =E<gt> 1 })>.
  357: 
  358: =item B<No mechanism to mark a C<$default> argument as optional>
  359: 
  360: When C<$default> is a string and zero arguments are received, the function
  361: always confesses.  There is no way to express I<"accept zero args and return
  362: undef gracefully">.
  363: 
  364: =item B<Positional-names C<$default> silently discards extra arguments>
  365: 
  366: C<get_params([qw(a b)], 1, 2, 3)> returns C<{ a =E<gt> 1, b =E<gt> 2 }>
  367: and ignores C<3>.  If strict arity is required, validate the returned hash
  368: with L<Params::Validate::Strict>.
  369: 
  370: =back
